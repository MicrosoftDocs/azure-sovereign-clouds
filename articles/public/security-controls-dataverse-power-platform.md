# Security controls in Dataverse and Power Platform

Security is the foundation of sovereignty. To ensure a successful deployment of Microsoft Sovereign Cloud solutions in Dataverse or Power Platform environments, review the following resources to understand the security, privacy, and sovereignty features available to support sovereign requirements for data residency and access control.

For an overview of the security controls that can help you protect your data and prevent unauthorized access to Dataverse, see [Protect your data with Dynamics 365 security controls - Dynamics 365 \| Microsoft Learn](/dynamics365/guidance/implementation-guide/security-strategy-security-controls).

[Datacenter security](/dynamics365/guidance/implementation-guide/security-strategy-security-controls) describes how Azure regional data centers physically protect your data from external and internal threats.

Follow the [secure-by-design guidelines and controls](/dynamics365/guidance/implementation-guide/security-strategy-security-controls) and use [best practices to secure and govern your Microsoft Power Platform environments](/training/paths/best-practices-environments/) to start your sovereignty journey from a secure baseline.

For more guidance in setting up your Dataverse and Power Platform environments to be both secure and sovereign, review these resources:

* [Data storage and governance in Power Platform](/power-platform/admin/security/data-storage)
* [What is Microsoft Dataverse?](/power-apps/maker/data-platform/data-platform-intro)  
* [Security concepts in Microsoft Dataverse](/power-platform/admin/wp-security-cds) 

To get more specific insights on secure implementation of Microsoft Dynamics, see the [Updated Scalable Security Modeling white paper](https://www.microsoft.com/download/details.aspx?id=45905). Dataverse and Power Platform provides fine-grained and multilevel access control that can help administrators ensure that their users and applications comply with sovereignty requirements. These controls are described in this article.
